使用加密的 `.env` 文件说明

概述
- 为避免在仓库中存放明文环境变量，我们可以将 `.env` 文件用对称加密保存为 `.env.enc`，并将 `.env` 保持为本地解密后的临时文件（并列入 `.gitignore`）。

加密 / 解密（需要系统安装 OpenSSL）

示例命令：

加密当前本地 `.env`（生成 `.env.enc`）：

```bash
ENV_PASSPHRASE="your-strong-passphrase" npm run env:encrypt
```

解密 `.env.enc`（恢复 `.env`）：

```bash
ENV_PASSPHRASE="your-strong-passphrase" npm run env:decrypt
```

注意
- 请使用强口令并通过安全渠道（例如系统 Secrets 管理器、CI secret）向运行环境提供 `ENV_PASSPHRASE`。不要把口令写入仓库。
- 完成加密并确认 `.env.enc` 可用后，本地应删除或清理明文 `.env`。
- 将 `.env.enc` 提交到仓库以便在其他环境中解密使用。

示例工作流程
1. 在本地创建并验证 `.env`。
2. 运行加密命令生成 `.env.enc`。
3. 确认 `.env.enc` 后，删除本地 `.env`（或将其留在本地但不提交）。
4. 在 CI 或其他机器上，使用受保护的 `ENV_PASSPHRASE` 解密得到 `.env` 并运行部署脚本。
